Classes and objects

The two ideas everything else in object-oriented programming is built on: a blueprint, and the real things made from it.

class — the blueprint, defined once
object — a real thing made from the blueprint

Example 1 — a car blueprint

A car manufacturer doesn't build a car for every customer from scratch. It designs one blueprint, then stamps out many actual cars from it.

the class

Car blueprint

Defines that every car will have a brand, a color, and a speed — and that every car can accelerate. The blueprint itself cannot drive anywhere; it's just the design.

the objects

Actual cars

Each car built from the blueprint has its own brand, its own color, and its own current speed. Changing one car's speed never affects any other car.

car1

brand = "Toyota"

color = "Red"

speed = 0

car2

brand = "Honda"

color = "Blue"

speed = 0

class Car: # this runs once, when a new Car object is created def __init__(self, brand, color): self.brand = brand self.color = color self.speed = 0 def accelerate(self, amount): self.speed = self.speed + amount # two separate objects, made from the same class car1 = Car("Toyota", "Red") car2 = Car("Honda", "Blue") car1.accelerate(40) print(car1.brand, car1.color, car1.speed) print(car2.brand, car2.color, car2.speed)
Notice: car1 sped up, but car2 stayed at 0. Each object keeps its own copy of brand, color, and speed — the class only defines what those attributes are called and how accelerate works, not their actual values.

Example 2 — a student record

A school doesn't write separate code for every student. It defines one Student class, then creates one object per real student.

the class

Student blueprint

Defines that every student has a name and a list of grades, and that every student can have a grade added and an average calculated.

the objects

Actual students

Asha and Ben are both Student objects. Asha's grades and Ben's grades live separately, even though they were built from the exact same class.

asha

name = "Asha"

grades = []

ben

name = "Ben"

grades = []

class Student: def __init__(self, name): self.name = name self.grades = [] def add_grade(self, grade): self.grades.append(grade) def average(self): if len(self.grades) == 0: return 0 return sum(self.grades) / len(self.grades) # two separate student objects asha = Student("Asha") ben = Student("Ben") asha.add_grade(90) asha.add_grade(85) ben.add_grade(70) print(asha.name, "average:", asha.average()) print(ben.name, "average:", ben.average())
Notice: calling add_grade on asha never changed ben's grades. The method add_grade is shared from the class, but the grades list it operates on belongs to whichever object called it.

Recap

  • A class is a blueprint. It is written once and describes what attributes and methods every object made from it will have.
  • An object is one real instance made from a class. Creating an object is sometimes called instantiating the class.
  • Attributes are the data each object stores, like brand, color, or grades. Each object has its own separate values.
  • Methods are the actions an object can perform, like accelerate or add_grade. The code is shared, but it always acts on the one object that called it.
  • self inside a class refers to "whichever object is currently running this method" — it's how a shared method finds the correct object's own data.
